# OSS Friction Log

Every bug, surprising behavior, or missing doc we hit while building/publishing Atlas
gets an entry here. Each entry is a candidate upstream contribution (issue + repro,
ideally with a PR). This is the pipeline for authentic open-source contributions —
original discoveries, not farmed "good first issues".

Rules:
- Log it the moment it happens, even one line. Polish later.
- An entry graduates when it has: minimal repro → upstream issue filed → (optionally) PR.
- One upstream contribution per week is the target cadence.

Upstream surfaces Atlas touches (watch these for friction):
- `vsce` / VS Code Marketplace publishing
- `ovsx` / Open VSX publishing
- VS Code extension APIs (webview, SecretStorage, contributes)
- React Flow (canvas)
- MCP TypeScript SDK + client config formats (Claude Code, Cursor, Windsurf, Gemini CLI, Codex)
- Zod / YAML parsing
